в Linux есть стандартные утилиты, которые помогут вам очистить систему и проверить файловую систему на наличие ошибок. 
Вот несколько полезных команд.

1. Очистка дискового пространства
Эти команды помогут удалить ненужные файлы, которые накапливаются со временем. Выполняйте их в терминале.

Удаление ненужных пакетов: Эта команда удаляет пакеты, которые были установлены автоматически как зависимости, но больше не требуются.
sudo apt autoremove

Очистка кеша пакетов: Эта команда удаляет скачанные файлы пакетов (.deb) из кеша apt.
sudo apt clean

Очистка логов системы (journald): Системные логи могут занимать много места. Эта команда ограничит их размер, например, до 100 мегабайт.
sudo journalctl --vacuum-size=100M

Поиск больших файлов и папок: Чтобы понять, что именно занимает место, можно использовать утилиту du. 
Эта команда покажет 10 самых больших файлов и папок в корневой системе:
sudo du -ah / | sort -rh | head -n 10
Вы можете изменить / на любой другой путь, например, на вашу домашнюю директорию (~), чтобы найти большие файлы там.

2. Проверка и исправление файловой системы
Для проверки файловой системы используется утилита fsck.

Важно: Проверку файловой системы (особенно с исправлением ошибок) нельзя запускать на примонтированном разделе. 
Поскольку вы хотите проверить корневой раздел (/), самый безопасный способ — запланировать проверку на следующую перезагрузку.

Запланировать проверку при следующей загрузке:
Выполните эту команду в терминале. Она создаст специальный файл forcefsck в корневом каталоге.

sudo touch /forcefsck

Перезагрузите Raspberry Pi.
sudo reboot

Во время загрузки система обнаружит файл /forcefsck, запустит проверку файловой системы и, если найдет ошибки, попытается их исправить. 
После этого файл будет автоматически удален.
